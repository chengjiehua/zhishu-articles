### Anthropic 即将迎来史上首个盈利季度

**来源：** Engadget / The Wall Street Journal / CNBC / Bloomberg
**链接：** https://www.engadget.com/2178340/anthropic-first-profitable-quarter/

Anthropic 正在接近一个历史性里程碑：据知情人士透露，这家 Claude 背后的公司预计 2026 年 Q2 营收将达 **109亿美元**，是 Q1（48亿）的两倍多，有望录得 **5.59亿美元运营利润**——这将是 Anthropic 自 2021 年成立以来的第一个盈利季度。

值得注意的是，Anthropic 同时正在完成一轮 300亿美元融资，估值冲向 **9000亿美元**，已经超过 OpenAI。公司已聘请 Wilson Sonsini 律所评估 IPO 路径。

但公司也坦承：这个季度的盈利不意味着后续季度也能持续盈利。AI 行业烧钱换规模的逻辑并未改变。

**编辑点评：** Anthropic 从「非营利实验室」到「9000亿估值的盈利企业」，只用了5年。但盈利的同时，这家公司也在和五角大楼就 AI 安全护栏问题公开对峙。盈利与原则之间的张力，会是接下来最值得观察的主线。

### 星巴克 AI 库存系统 9 个月即被废弃

**来源：** Engadget / Reuters / CNBC
**链接：** https://www.engadget.com/2179029/starbucks-abandons-its-ai-inventory-tool-after-only-nine-months/

星巴克本周正式废弃了部署仅 **9 个月**的 AI 自动化库存盘点系统。该系统覆盖北美门店的牛奶等饮品原料的库存管理，是 CEO Brian Niccol 解决长期缺货问题的核心举措之一。

但现实很骨感——系统频繁**标错和漏数商品**，例如将不同种类牛奶混淆。Reuters 获取的内部通讯显示，公司周一发布通知：「即日起，自动盘点将退役」。在一月的深度报道中，Reuters 已揭露该系统存在 AI 计数错误、供应商分散等问题。

**编辑点评：** 这是2026年迄今为止最具警示意义的企业AI落地案例。星巴克不是缺钱缺技术的小公司，但AI部署仍然翻车了。问题不在于模型不够聪明，而在于现实世界的数据噪音和运营复杂度远超实验室环境。那些还在画「AI全自动运营」PPT的企业，该读读这则新闻。

### AMD 发布 Ryzen AI Halo 迷你PC，定价 $3,999

**来源：** Engadget / Tom's Hardware
**链接：** https://www.engadget.com/2177687/amd-prices-its-ryzen-ai-halo-pc-at-dollar3999-unveils-ryzen-ai-max-400-chips/

AMD 正式公布 Ryzen AI Halo —— 一款 Mac Mini 大小的 AI 桌面工作站，**起价 $3,999**，6月开启预购。当前版本搭载 Ryzen AI Max 300 系列芯片。

同时，AMD 发布了新一代 **Ryzen AI Max 400（代号 Gorgon Halo）** 芯片，旗舰型号 AI Max+ Pro 495 为 16核/5.2GHz/55 TOPS NPU，支持最高 **192GB 统一内存**和 160GB GPU 显存。

### 1. multica-ai/andrej-karpathy-skills · ⭐ 142,923

**🔗** https://github.com/multica-ai/andrej-karpathy-skills

**它是什么：** 一个**单一的 CLAUDE.md 文件**，将 Andrej Karpathy 对 LLM 编码陷阱的观察提炼为四条行为准则。通过 Claude Code 插件或直接放入项目根目录即可生效。

**四大原则：**
- **先想再写** — 陈述假设，呈现多种解读，敢于 push back，遇到困惑停下来问
- **简单至上** — 没有多余功能、没有单次使用的抽象、没有未被要求的「灵活性」
- **精确修改** — 只碰必须碰的代码，不「顺便优化」相邻代码，匹配现有风格
- **目标驱动** — 用可验证的成功标准替代模糊任务描述，如「写测试→跑通→才算完成」

**技术栈：** Markdown / Claude Code Plugin 格式

**适合谁：** 经常用 Claude Code 写代码的开发者。一个文件解决 AI 写代码最常见的四大毛病：过度抽象、自说自话、乱改无关代码、目标模糊。

---

### 2. anthropics/claude-plugins-official · ⭐ 22,112

**🔗** https://github.com/anthropics/claude-plugins-official

**它是什么：** Anthropic 官方维护的 **Claude Code 插件目录**，包含内部插件（如 example-plugin 参考实现）和第三方社区插件。每个插件遵循标准结构（plugin.json + 可选的 MCP 配置 + commands/agents/skills）。

**核心功能：**
- `/plugins` 目录收录 Anthropic 内部开发的插件
- `/external_plugins` 目录收录合作伙伴和社区贡献的插件
- 通过 `/plugin install {name}@claude-plugins-official` 一键安装
- 第三方插件需通过质量与安全审核才能收录

**技术栈：** Claude Code Plugin 框架

**适合谁：** Claude Code 用户，尤其需要 GitHub/Figma/Context7 等工作流集成的开发者。

### 3. colbymchenry/codegraph · ⭐ 13,051

**🔗** https://github.com/colbymchenry/codegraph

**它是什么：** 预索引的**代码知识图谱**，为 Claude Code、Codex、Cursor、OpenCode 等 AI 编码代理提供零文件读取的代码探索能力。代理不再用 grep/glob/Read 扫描文件，而是直接查询符号关系、调用图和代码结构。

**实测数据（7 个真实开源项目，Claude Opus 4.7，4次取中位数）：**
- 平均降低 **35% 成本**，节省 **59% tokens**
- 平均快 **49%**，减少 **70% 工具调用**
- 25,874 文件的项目 **4 分钟完成索引**

**技术栈：** 自打包运行时（无需 Node.js），MCP 服务器

**适合谁：** 在大中型代码库中使用 AI IDE 的开发者。项目越大，效果越显著。

### 4. dotnet/skills · ⭐ 2,147

**🔗** https://github.com/dotnet/skills

**它是什么：** 微软 .NET 团队官方出品的 **AI 编码代理技能集**，遵循 AgentSkills.io 开放标准。覆盖 12 个插件领域：ASP.NET Core、Entity Framework、MAUI、MSBuild、NuGet、.NET 升级迁移、AI/ML 集成、模板引擎、测试诊断等。

**亮点：**
- 每个 Skill 都是标准化指令，直接注入 AI 代理上下文
- 附带有准确性/效率评分的 Dashboard（dotnet.github.io/skills）
- 支持 Claude Code、GitHub Copilot、Cursor、OpenAI Codex

**技术栈：** AgentSkills.io 标准，C# / .NET 生态

**适合谁：** .NET/C# 开发者。当 AI 代理不懂 ASP.NET 中间件模式或 EF Core 迁移时，给它装上这些 Skill。

### 5. antoinezambelli/forge · ⭐ 1,474

**🔗** https://github.com/antoinezambelli/forge

**它是什么：** Python 自托管 LLM **工具调用可靠性层**。通过 guardrails（救援解析、重试引导、步骤强制）和上下文管理（VRAM感知预算、分层压缩），将一个 8B 本地模型提升到多步代理工作流的顶尖水平。

**关键数据：**
- 当前最佳配置（Ministral-3 8B Q8 on llama-server）：**26 场景评测套件 86.5% 得分**，最难层级 76%
- 三种使用方式：WorkflowRunner（完整代理循环）、Guardrails 中间件（嵌入自己的编排）、Proxy（OpenAI兼容代理层）
- 支持 Ollama、llama-server、Llamafile、Anthropic 四种后端
- 附带 IEEE 学术论文级方法论

**技术栈：** Python 3.12+，Pydantic，asyncio

**适合谁：** 需要本地运行、不想调用付费 API 但又要可靠工具调用的开发者。预算有限但质量不妥协。
