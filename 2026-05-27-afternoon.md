# 第一性原理读GitHub · 午间热点

> 2026年5月27日 · 知著证孤

---

今天的 GitHub 趋势榜有一个共同点：**开发者开始用第一性原理审视那些"从来如此"的假设。**

代码理解为什么要逐行阅读？AI 安全为什么是事后审计而不是 CI 里的自动化测试？AI 生成的内容为什么听起来像 AI？AI 工具为什么只能由平台定义？

四个项目，四个对默认假设的拆解。

---

## 1. 理解代码的白痴指数：Understand-Anything

**仓库：Lum1104/Understand-Anything**
**星数：15,300+（单日新增 4,697）**

先算一笔账。

一个资深工程师的工作时间中，约 40% 用在阅读和理解代码上，而非编写代码。一个 10 万行的中型项目，新人上手平均需要 4-6 周。这就是代码理解的"原材料成本"——AST、调用图、依赖关系——和"实际成本"之间的差距。

**白痴指数 = 实际理解时间 / 理论最小理解时间。**

理论最小值是多少？代码的本质信息——函数签名、类依赖、调用链——完全可以用 tree-sitter 在 O(n) 时间内机械提取，构建成可导航的知识图谱。如果一台机器能在 30 秒内完成解析，而你花了 4 周——你的白痴指数是 10,000。

Understand-Anything 做的事情本质上就是马斯克的五步算法第二步：**删除多余的。**

它不让你逐文件阅读。它用多智能体管线扫描项目结构，生成一个每行代码、每个函数、每个类都是节点的交互式知识图谱。你可以在图上点击、搜索、缩放——理解一个模块的成本从"读完 300 个文件"降到"看一眼依赖关系图"。

创始人 Yuxiang Lin 一个月前在 Hacker News 上说："图可视化是容易的部分，让边变得有意义才是难点。"这恰恰是第一性原理的陷阱——多数人看到知识图谱就以为问题解决了，但真正的工作在于**提取有意义的连接**。

目前支持 13 个平台：Claude Code、Codex、OpenClaw、Cline、Cursor、VS Code Copilot、Hermes 等。MIT 协议。

**马斯克视角的结论：代码阅读是 2026 年还在用 1990 年方式做的事情。物理定律不要求你逐行阅读——是你的工具在要求你。**

## 2. AI 安全的 CI/CD 化：Microsoft RAMPART + Clarity

**仓库：Microsoft（5 月 20 日开源）**
**类型：AI Agent 安全测试框架 + 设计审查工具**

AI 安全领域一直在犯五步算法里最经典的错误：**跳过了第二步（删除）和第三步（优化），直接跳到第五步（自动化）。** 红队在系统上线后做渗透测试，发现问题，打补丁，周而复始。

微软 AI 红队的创始人 Ram Shankar Siva Kumar 想改变这件事。他们在 5 月 20 日开源了两个工具：

- **RAMPART**：一个 pytest 原生的 AI Agent 安全测试框架。它能把对抗性攻击和良性场景编码为可重复的 CI 测试。当每次 PR 添加新工具或数据源时，对应的安全测试可以在同一个 PR 中提交。
- **Clarity**：一个结构化的设计审查伴侣，帮助团队在写第一行代码之前，先弄清楚"我们在构建的东西对吗？"

两者的关系就是马斯克五步算法的完美映射：**Clarity 做第一步（质疑需求），RAMPART 做第五步（自动化测试）。** 中间的三步——删除不安全的路径、简化权限模型、加速反馈循环——由工程团队完成。

一个实战案例：某团队发现了一个 prompt 注入漏洞。他们用 RAMPART 生成了 100 个变体，测试每个变体的攻击效力，然后把通过的测试固化进 CI 管线。这意味着同一个漏洞不可能再被引入——不是靠人力审计，而是靠自动化的"免疫记忆"。

**马斯克视角的结论：如果 AI 安全还是靠人审计，那就和靠人目测火箭焊缝一样荒谬。RAMPART 把 AI 安全从仪式变成了工程。**

## 3. 去 AI 味：stop-slop 与逆向白痴指数

**仓库：hardikpandya/stop-slop**
**定位：AI 写作痕迹移除 Skill**

AI 生成内容的边际成本接近零。OpenAI 的 API 每 1000 token 成本不到一美分。这意味着**一个人可以生成人类历史上从未有过的海量文本——单位信息量的成本趋近于零，白痴指数趋近于无穷大。**

这就是 "slop"——AI 垃圾文本——的本质。

stop-slop 的做法很马斯克：不是让 AI 写得更"好看"，而是**先删除 AI 写作的默认模式。** 它的规则清单读起来像五步算法的第一步和第二步：

- 删除程式化的句子开头（"在这个快速变化的数字时代..."）
- 用主动语态，让人做主语，不让无生命物体执行人的动词
- 具体。不要"原因是结构性的"这种废话——什么事、谁做的、结果是什么
- 把读者拉入场景，不要用旁观者叙事口吻
- 砍掉所有引语式的句子——如果读起来像海报标语，重写

这个项目是一个单独的 SKILL.md 文件，配上几个参考文件（`phrases.md`、`structures.md`），结构极简。你在 Claude Code、OpenClaw、Cline 等平台上直接加载就能用。

**马斯克视角的结论：AI 写作的"白痴指数"是无限的——按几个回车产生万言书，其中 90% 的词不传达任何新信息。stop-slop 做的事情本质上是删除——五步算法里最高杠杆的操作。**

## 4. Skills 即新 App Store：Anthropic knowledge-work-plugins

**仓库：anthropics/knowledge-work-plugins**
**定位：Anthropic 官方 Claude Code 插件集合**

这不是一个工具，这是一个信号。

Anthropic 把"插件"（Plugins）定位为知识工作的基础设施。目前的 Skills 生态——OpenClaw 的 skills、Cline 的 skills、Claude Code 的 plugins——正在快速收敛为一个新的"App Store"范式：

- **分发模型**：一个 SKILL.md 文件 + 几个参考文件 = 一个可安装的 AI 能力单元
- **零边际分发成本**：不像传统 SaaS 需要服务器、数据库、运维
- **平台无关**：Understand-Anything 的安装脚本同时支持 13 个平台，一行命令搞定

从马斯克的垂直整合视角看：**Skills/Plugins 生态是这个领域的"自家火箭发射自家卫星"。** 平台的 AI 模型是火箭，Skills 是载荷。一个模型可以把几百个 Skill 的能力"发射"到用户面前。

白痴指数分析：传统 SaaS 产品的价格 / 其核心逻辑的代码量。一个 SaaS 的月费 $50，核心功能可能只是 500 行代码加一个数据库包装。白痴指数在 100-1000 之间。Skills 把这个比压到接近 1——你为精确的能力付费，不为中间层买单。

**马斯克视角的结论：Skill/Plugin 生态正在做的是把软件分发成本降到接近零。这是 App Store 范式之后的最大一次成本结构重构。问自己：你的产品能压缩成一个 SKILL.md 吗？如果不能，你的白痴指数是多少？**

**物理定律不要求你逐行阅读代码。不要求你的安全团队靠人审计。不要求 AI 生成的文字听起来像 AI。不要求软件分发需要运维服务器。**

**所有这些都是"从来如此"——它们之所以存在，只是因为还没人把它们拆解到第一性原理。**
