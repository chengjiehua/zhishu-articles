## 📊 本周 TOP 热榜

本周 GitHub AI 赛道延续高烧态势，**Agent 框架**和**自进化技能系统**成为绝对主线。以下是编辑精选的 6 个最值得关注的项目：

### 🥇 Hermes Agent — 141K ⭐ | 自进化 AI Agent

**NousResearch/hermes-agent** | MIT 协议 | 7 周破 10 万星

本周最炸裂的新面孔。Hermes 的核心理念是「用得越久越聪明」——它会根据你过去 10 次会话的实际表现**自动重写技能文件**，用经验驱动替代手工调参。

| 维度 | 特点 |
|------|------|
| 🧠 记忆系统 | 结构化经验存储，不是简单的 BM25/向量搜索 |
| 🔄 模型无关 | 同一套 Skill 可在 Claude / GPT / Llama 上跑 |
| 💬 多端接入 | Telegram、Slack、WhatsApp、Discord、Signal |
| 📈 增速 | 7 周 100K ⭐，超越 LangChain 和 AutoGPT 的历史记录 |

**编辑点评：** Hermes 的「技能自进化」理念是 Agent 框架的真正差异化方向。OpenClaw 的 Skills 体系走的是社区共享路线，Hermes 走的是个人经验积累路线——两者不矛盾，甚至可以组合使用。

---

### 🥈 Everything Claude Code — 140K ⭐ | Agent 性能调校系统

**affaan-m/everything-claude-code** | Anthropic 黑客松冠军

这不是一个 Agent，而是让 Claude Code / Cursor / Codex / OpenCode 四个主流编码工具**行为一致**的配置框架。

核心数据：28 个专用 Agent、119 个技能、60 个斜杠命令、170+ 贡献者、12 个语言生态。

```
v1.9.0（2026-03）亮点：
✅ 选择性安装架构（只装你需要的模块）
✅ 6 个新 Agent（含 TypeScript/PyTorch/Java Reviewer）
✅ AgentShield 安全扫描
✅ Cross-platform Hook 适配器
```

**编辑点评：** 对于团队来说，ECC 解决了「不同 Agent 产出质量不稳定」的痛点。如果你在用 AI coding agent 做生产级开发，这个仓库基本是必修课。

---

### 🥉 OpenHuman — 8.6K ⭐ | 私人 AI 超级智能

**tinyhumansai/openhuman** | Rust + TypeScript | GPL-3.0

与云端 Agent 不同，OpenHuman 主打**本地运行、隐私优先**。它是一个桌面应用（macOS/Windows/Linux），用强记忆系统「摄入」你的个人数据，构建专属的数字分身。

特点：完全离线可用，不依赖任何云端 API（可选接入），用 Rust 构建以保证性能。

**编辑点评：** 隐私 AI 是 2026 年的暗线。在「所有数据上云」的叙事之外，OpenHuman 代表了另一条路径——你的数据，你的模型，你的 Agent。

---

### 🏅 ML Intern — 9.5K ⭐ | 自主 ML 研究员

**huggingface/ml-intern** | Hugging Face 官方出品

能自主读论文、写代码、跑实验、交 PR 的 AI 实习生。基于 HuggingFace 的 smolagents 框架构建，深度集成 HF 生态系统（模型、数据集、论文、云端算力）。

```bash
# 一句话让它干活
ml-intern --prompt "复现这篇论文的结果并在 MNLI 上验证"
```

**编辑点评：** HuggingFace 正从「模型超市」转型「AI 劳动力平台」。ML Intern 是这个野心的第一步。

---

### 🏅 Pi Mono — 43.9K ⭐ | Agent 工具包全家桶

**badlogic/pi-mono** | MIT 协议

一个「什么都有」的 Agent 单体仓库：coding agent CLI、统一 LLM API、TUI/Web UI 库、Slack Bot、vLLM Pod 管理——全部打包在一个仓库里。

适合场景：当你不满足于「调个 API 就行」，想要完整的 Agent 运行时环境。

---

### 🏅 AgentMemory — 5K ⭐ | AI Agent 持久记忆

**rohitg00/agentmemory** | 基于真实基准测试

号称「#1 持久记忆方案」，专门解决 AI coding agent 跨会话丢失上下文的问题。基于真实世界基准测试优化，集成 MCP 协议。

## 🔭 趋势观察

**本周三个显著信号：**

1. **自进化 > 手动配置** — Hermes 和 ECC 都在证明：Agent 应该自己学会变好，而不是靠人不断写 prompt
2. **本地化浪潮起势** — OpenHuman、pi-mono 的本地部署方案受到关注，隐私+成本双驱动
3. **记忆成为核心战场** — AgentMemory、NeoCortex 等项目扎堆出现，上下文管理不再是一个 feature，而是一个独立赛道
