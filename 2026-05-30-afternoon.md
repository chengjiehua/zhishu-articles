# 白痴指数 · 午间 GitHub 热点

**2026 年 5 月 30 日 | 知著证孤**

---

今天的热点项目有一个共同主题：**去中间层**。有人用纯 C++ 重写了 Python 推理栈、有人把 TypeScript 从 JS 运行时里解放出来、有人用 9 美元 ESP32 替代了整套摄像头系统。这不是巧合——这是渐进极限法在多个领域的收敛。拆掉中间商，逼近物理极限。以下是五个最值得用工程透镜审视的项目。

---

## 1. tiny-vllm：推理引擎的「去 Python 化」实验

**仓库**：`jmaczan/tiny-vllm` | **本周关注**：42.5k+ stars | **语言**：C++ / CUDA

**结论先行**：Python 在推理栈里收了一道「解释器税」，tiny-vllm 就是这张账单。

vLLM 是当前最主流的 LLM 推理引擎，但它的架构是「Python 主体 + 预编译 CUDA 二进制」的混合体——调度器、请求队列、KV cache 管理大量跑在 Python 侧。小 batch、低延迟场景下，GIL 和解释器开销是可以测到的。社区关于「vLLM 的 Python overhead」已经吵了一年多。

一位独立开发者用纯 C++ 和 CUDA 从零重写了核心：PagedAttention、Continuous Batching，全部用 C++ 调度，CUDA kernel 直接调，没有任何 Python 中间层。代码量保持在数千行，定位在 Karpathy 的 `llm.c`（纯教学）和 vLLM（十几万行工业代码）之间的空白地带。

**五步算法视角**：这是第一步「质疑需求」的经典案例。为什么推理引擎需要 Python？不是因为 Python 带来了什么独特价值——而是因为 Python 生态方便开发。对生产环境而言，「方便开发」不是一个合理的需求。删掉它。

**渐近极限**：一块 GPU 的算力有其物理上限。任何推理引擎的延迟下限 = 模型计算量 ÷ GPU FLOPS。Python 层的存在让实际延迟偏离这个下限。tun-vllm 要做的就是逼近这个物理值。

**白痴指数**：如果定义「推理效率 = 实际吞吐量 ÷ 理论最大吞吐量」，vLLM 在 Python 层上大概损失 15-30%。这不是小数字——这意味着每三台 GPU 服务器就有一台在给解释器打工。

## 2. PerryTS：TypeScript 不需要 JavaScript 运行时

**仓库**：`PerryTS/perry` | **本周关注**：28.4k+ stars | **语言**：Rust / LLVM

**结论先行**：TypeScript 的类型系统是编译信息，JavaScript 运行时是历史包袱。把它们焊在一起是行业惯性，不是物理必然。

Perry 是一个用 Rust 写的 TypeScript 编译器。它把 `.ts` 文件通过 SWC 解析、直接生成 LLVM IR，然后编译成原生二进制——没有 Node.js，没有 V8，没有 Electron。输出是单个可执行文件，零运行时依赖。

基准测试数据：fibonacci 309ms（对标 Rust 316ms、C++ 309ms），object create 与 Rust/Swift 同级，data-dependent loop 225ms（Rust 226ms）。原生性能，TypeScript 语法。

**第一性原理检验**：TypeScript 的静态类型系统本身携带足够的编译时信息来做优化。JS 运行时（V8/Node.js）做的 JIT 编译本质上是「在运行中重新发现编译时已知的信息」。这是冗余。Perry 把 JIT 变成 AOT——在编译时就利用好类型信息，不等到运行时再做推断。

**五步算法第二步**：删掉不必要的部分。V8 引擎几百万行 C++ 代码，其中大量逻辑是为动态类型脚本语言设计的——而 TypeScript 不是动态类型语言。Node.js 的事件循环、libuv、模块系统——对于「我要一个可执行文件」这个需求而言全是多余的。删。

**垂直整合含义**：如果把「从 TypeScript 到可执行文件」看作一条供应链，传统路线是：tsc 编译 → JS 文件 → Node.js/V8 运行时 → x86 指令。中间商（V8）抽取了大量性能和信息。Perry 垂直整合了这个链条：TS 源码 → LLVM → 原生指令。没有中间商。

## 3. RuView：9 美元的穿墙感知系统

**仓库**：`ruvnet/RuView` | **本周关注**：22.2k+ stars | **语言**：Rust

**结论先行**：监控摄像头的白痴指数高到荒谬。RuView 用物理学原理替代了硬件堆料。

这个项目把商用 WiFi 信号转化为实时空间感知——检测人体姿态（17 个关键点）、监测呼吸和心率、感知房间内人员存在。不需要摄像头，不需要穿戴设备，在黑暗中、穿墙都能工作。硬件成本：3-6 块 ESP32-S3 芯片，总价约 54 美元（最低 9 美元单节点）。

原理不新奇：WiFi 信号的 Channel State Information（CSI）会被人体的移动和呼吸微妙地扰动。CMU 在 2023 年发表了 DensePose from WiFi 论文，Verizon 2025 年推出了商用 WiFi 感知服务。新奇的是：RuView 的 AI 模型只有 8KB（4 位量化），在树莓派上微秒级运行，声称 100% 存在检测准确率。

**白痴指数**：
- 原材料成本（ESP32 芯片 + 天线 + PCB）：约 $3-4/节点
- 传统摄像头监控系统：$500-5000
- 白痴指数：**125-1250x**

这不是「技术创新」，这是物理必然。环境中已经充满了 WiFi 信号。人体扰动这些信号在物理上是不可避免的。所以「用 WiFi 感知人」不是「能不能做」的问题——是「为什么现在才做」的问题。

**质疑需求**：监控摄像头为什么需要「拍视频」这个功能？绝大多数安防场景只需要回答三个问题：有人吗？几个人？在动吗？视频流中 99% 的像素信息是冗余的。CSI 信号可以直接回答这三个问题而不产生任何隐私数据。这是五步算法的第一步和第二步的完美示范：先质疑需求，再删除多余的（视频流）。

## 4. TradingAgents：用 LLM 代理替代整支交易团队

**仓库**：`TauricResearch/TradingAgents` | **本周关注**：62.6k+ stars | **语言**：Python

**结论先行**：一支交易团队的人力成本大概是每年 300 万美元。LLM token 成本大约是每次分析 0.01 美元。这是 3 亿倍的差距。这就是白痴指数在知识工作领域的投射。

TradingAgents 把一个完整交易公司搬进了代码：基本面分析师、情绪分析师、技术分析师、多头/空头研究员、交易员、风险管理团队——每个角色是一个独立的 LLM 代理，有专属的系统提示和工具权限。它们辩论、投票、最终产出交易决策。论文报告年化收益率最高 30.5%，显著优于传统策略。

**第一性原理检验**：一个人类分析师读财报、看新闻、出判断的本质是什么？信息摄入 → 模式识别 → 决策输出。LLM 在信息吞吐量上远超人类（一分钟读完 100 份财报），在模式识别上正在逼近（金融文本的情绪分析已有成熟基准），在决策一致性上已经超过（不会因为昨晚没睡好而出错）。

**问题是**：回测 30.5% 年化不代表实盘 30.5%。文档记录的一次真实运行：30 天内 7% 收益（vs S&P 500 的 4.5%），但最大回撤 22%，可复现性未验证。这是快速迭代阶段的典型特征——东西能跑，但还没过「产能地狱」。

**五步算法预警**：当前版本在疯狂往框架里加功能——多数据源、多代理角色、多市场。别忘了第二步：先删。一个只有分析师和风险管理的两代理系统可能跑得比六代理完整版更好。复杂不等于智能。

## 5. MoneyPrinterTurbo：视频生产逼近零边际成本

**仓库**：`harry0703/MoneyPrinterTurbo` | **本周关注**：70.7k+ stars | **语言**：Python

**结论先行**：短视频制作的人力成本即将归零。这不是内容行业的危机，是内容供给侧的解放——五步算法第四步「加速」贯彻到了极致。

这个项目做了一件事：输入一个主题，输出一条完整高清短视频。脚本撰写、AI 生成画面、语音合成、背景音乐、后期合成——全流程自动化。README 写的是「零门槛，零剪辑经验」，演示视频确实来自一句话输入。

**渐近极限分析**：一条 60 秒短视频的物理最小「生产成本」是多少？脚本：LLM 生成成本 $0.0001。画面：AI 图像/视频模型推理成本 $0.01-0.05。语音：TTS 成本 $0.0005。音乐：生成式 BGM 成本 $0.001。合成：纯计算 $0.0001。**总计：约 $0.012-0.052/条**。

传统短视频团队报价：$500-5000/条。白痴指数：**1 万到 10 万倍**。

**制造 vs 设计**：这个项目的精髓不是「视频模型好」——而是把整个 pipeline 跑了。脚本→画面→配音→合成，每一步的质量不是最好的，但端到端能跑通。这比等一个完美的「视频版 GPT-5」出现更有价值。先跑起来，从反馈中迭代。SpaceX 前三次火箭都炸了，第四次才成功。MoneyPrinterTurbo 现在的视频可能不是奥斯卡级别，但 pipeline 已经在了——从这条 pipeline 里能学到的东西，比任何纸面设计都多。
